# Atlas Charts

Generated by `generate_atlas.py`. Each chart has two variants: raw scores and excess-corrected scores.

## Raw vs Excess Scores

**Raw scores** measure how much attention each head directs to each behavior type (delimiter positions, duplicate tokens, etc.). These are inflated by base rates: if 30% of positions are delimiters, a uniform-attention head scores 0.30 on delimiter without any specialization.

**Excess scores** subtract the step-0 base rate (measured from random init, where attention is approximately uniform). A head with 0.30 raw and 0.30 base rate has 0.00 excess (no specialization). A head with 0.30 raw and 0.10 base rate has 0.20 excess (genuine specialist). The excess methodology comes from the merge-barriers paper (Blackwell, 2026).

**Use excess charts for classification.** Use raw charts only for understanding the measurement methodology.

## Charts

### developmental-timeline-{raw,excess}.png

Stacked area chart showing the number of heads classified as each behavior type at every training step, for both baseline (standard BPE) and comparison (merge barriers) side by side.

**What to look for:** When do different specializations emerge? How does the distribution stabilize? The excess version shows the true developmental sequence: positional_prev dominates (not delimiter, which was inflated in raw).

### entropy-divergence.png

Mean attention entropy over training for both runs. No raw/excess distinction (entropy is computed directly from attention weights, not from behavior scores).

**What to look for:** Baseline entropy rises from ~0.35 back to ~0.70 in late training (heads becoming more diffuse). Comparison stays flat at ~0.35. The merge-barrier model maintains focused attention throughout training while the standard BPE model's attention degrades.

### p0-sink-emergence-{raw,excess}.png

Count of P0 sink (dormant) heads over training. P0 sinks are heads that route disproportionate attention to position 0 (the attention sink pattern from Xiao et al., 2024).

**What to look for:** Baseline develops ~27 P0 sinks at step 20000 (raw). Comparison develops ~5. The merge-barrier tokenizer prevents head dormancy. With excess correction, the counts change because the P0 classification threshold shifts, but the ratio remains similar.

### specialization-index-{raw,excess}.png

Mean specialization index (max score / sum of scores) over training. Higher = more heads are pure specialists. Lower = more heads are generalists.

**What to look for:** Both runs show similar specialization trajectories. The excess version is more informative because it removes the artificial inflation that makes raw scores look more specialized than they are.

### layer-depth-specialization-{raw,excess}.png

Heatmap of per-layer mean specialization index over training. Y-axis is layer (0-23), X-axis is training step, color intensity is specialization.

**What to look for:** Do early layers specialize first? Middle layers (8-12) show the strongest specialization increase in baseline. Comparison distributes specialization more evenly across layers. Layer 0 de-specializes over training (starts high, ends low).

### polysemanticity-{raw,excess}.png

Count of "specialist" heads (index > 0.7) and "generalist" heads (index < 0.3) over training.

**What to look for:** Most heads are moderate specialists (0.3-0.7), not extremes. The merge-barrier model has fewer extreme specialists AND fewer extreme generalists, pushing more heads into the capable-but-flexible range.

## Regenerating

```bash
python generate_atlas.py                     # light theme, raw scores
python generate_atlas.py --use-excess        # light theme, excess scores
python generate_atlas.py --both-scores       # both raw and excess
python generate_atlas.py --dark              # dark theme
python generate_atlas.py --both-themes       # both themes
```
